Scalars — the atoms of YAML

Every value in a config file is ultimately a scalar. Know them exactly.

⏱ ~15 min 🎯 Scalar types, quoting rules, gotchas for yaml.v3

What is a scalar?

A scalar is a single, indivisible value — a string, a number, a boolean, or null. Scalars are the leaves of every YAML document. Every key name is a scalar. Every value that isn't a list or a map is a scalar.

host:     localhost      # string scalar
port:     5432           # integer scalar
timeout: 30.5           # float scalar
debug:   true           # boolean scalar
secret:  #              <-- null scalar (empty value)

YAML infers the type of a scalar from its content — a process called implicit typing. The rules are baked into the spec and determine what Go type your yaml.v3 unmarshaller will produce.

Strings

A string is any scalar that doesn't look like a number, boolean, or null. No quotes needed in the common case:

name: my-cli-tool
description: A tool that does things

When you must quote

Quoting forces YAML to treat the value as a string, overriding implicit typing. You must quote when:

  • The value looks like another type: "true", "42", "null", "1.0e3"
  • The value contains special characters that begin a structure: :, #, [, {, ,
  • The value has leading or trailing spaces you want to preserve
  • The value starts with an indicator character: -, ?, |, >, !, &, *, %

Single quotes vs double quotes

Single quotes ' '

'It''s alive'      # '' escapes a '
# No backslash sequences
# What you see is what you get

Double quotes " "

"Line one\nLine two"
"Tab\there"
"Unicode: é"
# Backslash escapes work

Use single quotes when the string contains backslashes you don't want interpreted (Windows paths, regex patterns). Use double quotes when you need escape sequences like \n or \t.

Booleans — the biggest gotcha

YAML 1.1 (which many libraries implement, including older versions of Go's yaml) recognised a very wide set of boolean literals:

# All of these parsed as booleans in YAML 1.1:
true  True  TRUE
false False FALSE
yes   Yes   YES
no    No    NO
on    On    ON
off   Off   OFF
Gotcha for Go config files

yaml.v3 implements YAML 1.1 boolean rules. If your config has a key like mode: on and the Go field is a string, yaml.v3 will fail to unmarshal it — it sees a boolean, not a string. Quote it: mode: "on". Country codes like NO (Norway) are a classic victim of this rule.

YAML 1.2 (the current spec) restricts booleans to only true and false (case-sensitive). But until yaml.v3 fully migrates, treat yes/no/on/off as reserved words in config files.

Integers

YAML supports decimal, hexadecimal, and octal integer literals:

decimal: 42
hex:     0xFF          # 255
octal:   0o17          # 15  (YAML 1.2 octal)
old_oct: 017           # 15  (YAML 1.1 octal — leading zero!)
negative: -7
Warning

In YAML 1.1, a leading zero triggers octal parsing: port: 0777 becomes decimal 511, not 777. yaml.v3 inherits this. Always use 0o prefix for octal, or quote the value if you mean a string like "0777".

Floats

rate:       1.5
scientific: 1.5e3        # 1500.0
infinity:   .inf         # positive infinity
neg_inf:    -.inf
not_a_num:  .nan

These are valid YAML scalars and will unmarshal into Go float64. .inf and .nan map to math.Inf(1) and math.NaN() respectively.

Null

optional_field: null      # explicit null
also_null:      ~          # tilde is the null shorthand
empty_value:               # bare empty value is also null

In Go, null unmarshals to the zero value of the target type — nil for pointers, "" for strings, 0 for ints. Use a pointer type in your Go struct if you need to distinguish "not present" from "zero value".

Plain scalars — the implicit typing resolution order

When YAML sees an unquoted scalar, it tests it against this ordered list of patterns. The first match wins:

Null? → null | ~ | (empty) Boolean? → true | false | yes | no | on | off (+ variants) Integer? → [-+]?[0-9]+ | 0x[0-9a-fA-F]+ | 0[0-7]+ Float? → [-+]?(\.[0-9]+|[0-9]+\.[0-9]*)([eE][-+]?[0-9]+)? | [-+]?(\.inf|\.Inf|\.INF) | \.nan String → everything else

This is why true is a boolean — it matches the boolean pattern before string is reached. Quoting exits this resolution entirely.
