Putting It Together — deliberate config design

Build a real Go CLI config file. Every choice justified. No accidents.

⏱ ~20 min 🎯 Synthesis: block scalars + anchors + tags + strict loader in one config

The brief

You're designing the config file for a Go CLI tool called syncd — a data-sync daemon that reads from a primary database, writes to a replica, ships logs to a file, and supports optional plugins. The config needs to be:

  • DRY — database connection defaults shared, not duplicated
  • Safe — no implicit type surprises; ambiguous values are explicit
  • Readable — multi-line values are block scalars, not escaped strings
  • Strict — the loader rejects unknown keys and extra documents

We'll build the config file first, then the matching Go structs and loader. Every feature used is deliberate.

The config file

---   # explicit document start — signals "this is YAML" to readers

# ── Anchor-only defaults block ──────────────────────────────────────────
# Keys prefixed with _ are never read by Go (unknown keys ignored by default).
# Their sole job is to be merged elsewhere.
_defaults:
  db: &db-defaults
    port:            5432
    connect_timeout: 5
    read_timeout:    30
    ssl:             true

# ── App ─────────────────────────────────────────────────────────────────
app:
  name:    syncd
  version: !!str 1.0   # !!str: 1.0 would be a float without this

# ── Databases ───────────────────────────────────────────────────────────
primary:
  <<: *db-defaults      # inherit port, timeouts, ssl
  host: db-primary.internal
  name: syncd_prod
  password: !vault secret/syncd/primary-password

replica:
  <<: *db-defaults      # same defaults, different host
  host: db-replica.internal
  name: syncd_prod
  ssl:  false             # override: replica uses plain connection
  password: !vault secret/syncd/replica-password

# ── Log ─────────────────────────────────────────────────────────────────
log:
  level:   info
  outputs: [stdout, /var/log/syncd.log]   # flow: short flat list of scalars
  format: |-                              # strip: no trailing \n (feeds a template)
    {{.Time}} {{.Level}} {{.Msg}}
    {{- if .Fields}} {{.Fields}}{{end}}

# ── Plugins ─────────────────────────────────────────────────────────────
plugins:
  - name:    audit
    enabled: true
    config:
      retain_days: 90
  - name:    cache
    enabled: false

Every choice annotated

!!str 1.0 — 1.0 is a valid float; !!str forces "1.0" as a string &db-defaults — DRY: port/timeouts/ssl defined once, merged into primary & replica <<: *db-defaults — merge key pulls in defaults; explicit keys (ssl: false) override !vault — local tag signals "resolve this via Vault, not a plain string" [stdout, ...] — flow sequence: short, homogeneous, reads naturally inline |- — strip chomping: format string feeds a Go template; trailing \n is a bug sequence of maps — plugins are ordered and named; a mapping would lose order

The matching Go structs

type Config struct {
  App     AppConfig      `yaml:"app"`
  Primary DBConfig       `yaml:"primary"`
  Replica DBConfig       `yaml:"replica"`
  Log     LogConfig      `yaml:"log"`
  Plugins []PluginConfig `yaml:"plugins"`
}

type AppConfig struct {
  Name    string `yaml:"name"`
  Version string `yaml:"version"`  // !!str in YAML → string here, safe
}

type DBConfig struct {
  Host           string   `yaml:"host"`
  Port           int      `yaml:"port"`
  Name           string   `yaml:"name"`
  SSL            bool     `yaml:"ssl"`
  ConnectTimeout int      `yaml:"connect_timeout"`
  ReadTimeout    int      `yaml:"read_timeout"`
  Password       VaultRef `yaml:"password"`
}

type VaultRef struct { Path string }

func (v *VaultRef) UnmarshalYAML(node *yaml.Node) error {
  if node.Tag != "!vault" {
    return fmt.Errorf("password must use !vault tag, got %s", node.Tag)
  }
  v.Path = node.Value
  return nil
}

type LogConfig struct {
  Level   string   `yaml:"level"`
  Outputs []string `yaml:"outputs"`
  Format  string   `yaml:"format"`  // |- in YAML → no trailing \n, ready for template
}

type PluginConfig struct {
  Name    string                 `yaml:"name"`
  Enabled bool                   `yaml:"enabled"`
  Config  map[string]interface{} `yaml:"config,omitempty"`
}

The strict config loader

func Load(path string) (*Config, error) {
  f, err := os.Open(path)
  if err != nil {
    return nil, err
  }
  defer f.Close()

  dec := yaml.NewDecoder(f)
  dec.KnownFields(true)   // reject unknown keys

  var cfg Config
  if err := dec.Decode(&cfg); err != nil {
    return nil, fmt.Errorf("parse error: %w", err)
  }

  // Enforce single-document stream
  var extra yaml.Node
  if err := dec.Decode(&extra); !errors.Is(err, io.EOF) {
    return nil, fmt.Errorf("config must contain exactly one YAML document")
  }

  return &cfg, nil
}
KnownFields and _defaults

KnownFields(true) will reject the _defaults key because Config has no such field. Two fixes: (1) add Defaults interface{} \`yaml:"_defaults"\` as a discard field, or (2) move anchor definitions into a separate file that is never parsed by your Go loader and use a pre-processing step to concatenate it. The simplest production choice is option 1.

The complete picture

Here is how each YAML feature maps to a Go concern in this config:

YAML feature Go concern ───────────────── ──────────────────────────────────── !!str 1.0 Guarantees Version is a string, not float64 &db-defaults / << DRY — one source of truth for shared DB settings !vault Typed credential — VaultRef, not plain string |- (strip) No trailing \n — format string is template-ready [a, b] flow seq Readable inline — short flat list of output paths sequence of mappings Plugin order is defined and stable yaml.Decoder Safe multi-doc detection — no silent truncation KnownFields(true) Typos in key names become load errors, not silent misses
